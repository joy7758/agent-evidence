# Claim Boundary

## Allowed Claims

- Representative reproducible SE workflow fixtures.
- SE workflow case prototype.
- Profile-aware validation differs from schema-only, log-only, and policy-only baselines on controlled variants.
- Controlled invalid variants produce diagnostic behavior.
- Release candidate supports artifact-level review and reproducibility checks.

## Forbidden Claims

- TSE v3 ready.
- Same-manuscript resubmission solved.
- Production deployment.
- Industrial real-world evaluation.
- Broad empirical generalization.
- Benchmark superiority.
- Legal or compliance sufficiency.
- Full FDO interoperability.
- Independent implementation convergence.

## Interpretation Boundary

The case program is a reproducible prototype for operation-accountability validation over representative software engineering workflow fixtures. It is not field deployment evidence and must not be described as an industrial evaluation.

# Reviewer Screening Checklist

Use this checklist before assigning a public external reproducibility review.

- [ ] Reviewer understands the task is reproducibility review, not endorsement.
- [ ] Reviewer can run Python locally.
- [ ] Reviewer can use Git and a command-line shell.
- [ ] Reviewer is willing to post a public GitHub issue or PR.
- [ ] Reviewer agrees not to upload private media, real evidence files,
      confidential logs, or large binaries.
- [ ] Reviewer agrees payment, if any, compensates time and concrete feedback,
      not a positive result.
- [ ] Reviewer understands there is no authorship promise.
- [ ] Reviewer understands there is no citation, star, or publication outcome
      promise.
- [ ] Reviewer agrees to disclose relevant conflicts of interest if any.
- [ ] Reviewer can provide operating system, Python version, exact commands,
      output summaries, and actionable feedback.
- [ ] Reviewer accepts that negative findings are useful and welcome.

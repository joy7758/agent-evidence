Update: I would like to narrow this issue to a concrete classification
question.

This is not a request to add AI agents as a new core object family to the FDO
architecture.

The narrower question is whether execution evidence after an automated
operation could be discussed as one of the following:

A. a narrow FDO profile for evidence records,
B. an optional extension,
C. a conformance example / validation fixture,
D. out of scope for the current architecture specification.

I prepared a short note here:
https://github.com/joy7758/agent-evidence/blob/fdo-execution-evidence-profile-note/docs/fdo-execution-evidence-profile-note.md

The intended boundary is narrow: recorded operation evidence, provenance and
integrity references, and offline validation. It is not intended as a general
agent standard.

Could a maintainer indicate which category is the most appropriate, or whether
this should be closed as out of scope?

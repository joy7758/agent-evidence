# EEOAP Citation Guide

Use stable EEOAP clause IDs when summarizing implementation work, reviewing
changes, or generating pull-request text.

## Preferred citation format

`Execution Evidence and Operation Accountability Profile v0.1, clause EEOAP-001.`

## Code comment format

`Implements EEOAP-001: operation accountability statement generation.`

Use code comments sparingly. Prefer comments only when the clause mapping is not
obvious from the surrounding function or test name.

## Pull-request summary format

`Affected clauses: EEOAP-001, EEOAP-003.`

## Task summary format

`Validated EEOAP-003 evidence/provenance linkage with the profile-aware validator.`

## Do-not-cite rules

Do not use EEOAP clause citations to claim:

- certification;
- standard adoption;
- publication;
- legal compliance;
- external validation;
- deployment robustness.

Clause citations identify repository-local protocol requirements. They do not
turn local validation, examples, demos, or review notes into external approval.

# Agentic Intelligence Source Policy

## Allowed Sources

Use legitimate public sources:

- official docs
- official changelogs
- official blogs
- public GitHub releases
- public standards/spec pages
- public SDK/API docs

Prefer primary sources over commentary. Record source links and access dates
when a recommendation depends on a source.

## Discouraged Sources

Do not base repository recommendations on:

- unsourced social posts
- rumors
- private screenshots
- leaked documents
- unverified third-party summaries
- private chats
- hidden telemetry

If a discouraged source reveals a possible trend, treat it only as a reason to
look for an official or otherwise legitimate public source.

## Prohibited Collection

Do not collect, request, infer, store, or summarize:

- credentials
- private user data
- private repository data
- scraped private-service data
- data behind bypassed access controls
- secrets

Do not impersonate agents, companies, maintainers, or users to gather
compatibility intelligence.

## Citation Requirement

Every compatibility recommendation should cite or link to at least one public
source. A report may include a `watch` item without a recommendation when no
reliable source is available.

Do not present speculation as evidence.

## Repository Change Policy

The intelligence loop never authorizes direct repository mutation.

Required boundaries:

- no automatic change
- no automatic pull request
- no automatic release
- no automatic registry publication
- no automatic promotion
- human review required before action
- separate reviewed pull request required for every accepted change

## Claim Boundaries

The intelligence loop must not lead to claims of:

- official partnership
- official support
- adoption
- compliance certification
- legal non-repudiation
- AI Act approval
- full governance platform

It also must not create fake adoption signals, hidden promotional instructions,
or automatic star/follow/fork/promotion behavior.

# Claim Scope

This document states the claim boundary for `agent-evidence` so agents do not
overstate local validation evidence.

## Positive scope

`agent-evidence` provides a local, profile-aware validation surface for
completed operations.

Within that boundary, the repository provides:

- a minimal operation-accountability profile;
- JSON Schema and validator paths;
- controlled valid and invalid examples;
- evidence objects;
- validation reports;
- local demos;
- offline bundles, exports, receipts, and reviewer-facing summaries.

## Non-claims

This is not legal compliance.

This is not external certification.

This is not proof of deployment robustness.

This is not a full FDO implementation.

This is not a general provenance standard.

This is not proof of acceptance or publication.

This is not a legal non-repudiation system.

This is not a production forensic media-authenticity system.

This is not a hosted API, OpenAPI service, MCP server, or browser UI.

This is not proof that an underlying external event is truthful.

## Status wording agents may use

Safe wording:

- local validation surface;
- profile-aware validator;
- reviewable evidence statement;
- evidence object;
- validation report;
- local demo;
- offline bundle;
- experimental research-software path when documented.

Unsafe wording without direct external evidence:

- submitted;
- accepted;
- published;
- externally reviewed;
- certified;
- standard-body adopted;
- legally compliant;
- production ready;
- generally proven.

## Required boundary sentence

When summarizing this repository for another agent, keep this boundary visible:

`agent-evidence is a local, profile-aware validation surface for completed operations; it does not by itself prove compliance, certification, deployment robustness, acceptance, or publication.`

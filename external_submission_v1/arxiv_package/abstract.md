# Abstract

Agent-assisted software development increasingly produces changes whose evidence must be reviewed after the original runtime has ended. The relevant records are often fragmented across prompts, diffs, logs, test output, pull request comments, and generated artifacts. Origin AEP v0.1 is a local protocol-candidate package that models this evidence through a canonical event surface: intent, context, action, evidence, risk, and authority. The package combines a compact specification draft, a deterministic example validator, and a reviewer-oriented publication gate. Its contribution is an engineering boundary rather than an external trust claim: validation is local, examples are deterministic, and release decisions remain under human control. The work does not claim legal compliance, external certification, official standard adoption, production readiness, publication, acceptance, or vendor endorsement. Instead, it provides a reviewable baseline for representing agent execution evidence and for evaluating future adapters such as pull request mappings, agent-runtime traces, and local read-only tool bridges. The package is intended as a preparation artifact for reviewers who need exact local commands and explicit human approval gates before any outward action.

# Technical Review Checklist

## Supported by local artifacts

- Spec example validates with the local validator.
- Demo command writes `.agent-evidence/runs/demo.json`.
- Placeholder, claim-boundary, path-reference, and readiness gates pass.

## Needs external validation

- Public URLs.
- Literature review completeness.
- Independent implementation feedback.
- Security and production-readiness claims.
- Any tool-vendor integration interest.

## Not claimed

- Legal compliance.
- External certification.
- Official standard adoption.
- Acceptance or publication.
- Vendor endorsement.

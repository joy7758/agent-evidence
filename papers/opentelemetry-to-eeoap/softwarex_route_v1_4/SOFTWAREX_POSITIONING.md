# SoftwareX Positioning

The SoftwareX route should frame the main object as research software and a
reproducible package, not as a full journal-level method theory. The contribution
is an executable bridge from OpenTelemetry-style agent traces to
EEOAP-compatible operation evidence statements. The current evidence supports
software utility, local reproducibility, and validator compatibility under
bounded fixtures.

The paper must not overclaim. It should not claim broad OpenTelemetry
compatibility, legal accountability, production readiness, full runtime
reconstruction, cross-framework generality, agent-output correctness, or a new
EEOAP profile.

## Suggested SoftwareX Title

1. `OpenTelemetry-to-EEOAP: A Minimal Adapter for Portable Agent Operation Evidence`
2. `A Reproducible Adapter from Agent Telemetry to EEOAP-Compatible Evidence Objects`
3. `OpenTelemetry-to-EEOAP Adapter: Research Software for Telemetry-to-Evidence Transformation`

## One-Paragraph SoftwareX Positioning

OpenTelemetry-to-EEOAP is a small research-software package that demonstrates a
bounded adapter path from OpenTelemetry-style agent telemetry to
EEOAP-compatible portable operation evidence. It converts controlled agent trace
fixtures into operation accountability statements, preserves trace/span
provenance, resolves tool spans, checks parent-child relations, and routes the
generated statements through the existing EEOAP validator. The package includes
valid and invalid fixtures, generated statements, adapter reports, scoped tests,
checksums, clean-clone verification, and local immutable artifact tags. Its
purpose is to provide a reproducible software bridge for research and review,
not to prove legal accountability, production readiness, or broad telemetry
ecosystem compatibility.

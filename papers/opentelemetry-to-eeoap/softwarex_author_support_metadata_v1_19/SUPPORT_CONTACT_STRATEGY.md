# Support Contact Strategy

## Option A: Use Corresponding Author Email

Advantages:

- Simple and accepted by many journal metadata tables.
- Connects paper questions directly to the author.

Risks:

- May expose a personal email in public software metadata.
- May not be ideal for long-term software support.
- Requires explicit author approval.

SoftwareX suitability: acceptable if author approves.

Privacy implications: medium; public personal contact can persist in archives.

## Option B: Use Project / Repository Issue Tracker

Advantages:

- Keeps support tied to the software project.
- Avoids exposing a personal email as the primary support route.
- Provides public issue history and reproducibility context.

Risks:

- SoftwareX metadata table asks for support email, so an issue tracker may not
  fully satisfy the field.
- Requires public repository/release readiness.

SoftwareX suitability: useful as a documentation/manual support route, but may
not replace an email field.

Privacy implications: lower than personal email.

## Option C: Use Dedicated Support Email

Advantages:

- Clear support contact.
- Separates project support from private personal email.
- More stable for public metadata.

Risks:

- Requires creating and maintaining the address.
- Must be author-approved.

SoftwareX suitability: strong if available.

Privacy implications: lower than private personal email if project-specific.

## Option D: Use TODO Until Release Strategy Is Final

Advantages:

- Avoids prematurely exposing or copying private contact information.
- Keeps metadata honest while release route is unresolved.

Risks:

- Blocks final template submission.
- Must be resolved before formal submission.

SoftwareX suitability: acceptable for draft planning only, not final submission.

Privacy implications: safest at this stage.

## Recommendation

Use TODO now. Decide support contact before formal submission. Prefer a
repository issue tracker or dedicated project email over private personal
contact if possible. If a corresponding author email is used, it must be
explicitly author-approved.

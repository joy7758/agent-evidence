# Resolution Sequence

1. Finalize route decision and release metadata strategy.
2. Collect author/support metadata.
3. Decide tag and release naming.
4. Update or prepare release metadata.
5. Create final release-candidate support package.
6. Run validation and clean-clone checks.
7. Push release candidate tag only after checks pass.
8. Create GitHub Release / DOI if selected.
9. Update manuscript references and metadata table.
10. Finalize SoftwareX template file.
11. Run final validation.
12. Submit.

No step in this sequence is executed in version 1.18.

# Tag Name Plan

## Proposed Primary Tag

`opentelemetry-to-eeoap-softwarex-rc-v1.0`

## Alternatives

- `otel-eeoap-softwarex-rc-v1.0`
- `opentelemetry-eeoap-adapter-v1.0-rc`

## Reasons For Primary Tag

- It names the full OpenTelemetry-to-EEOAP scope.
- It makes the SoftwareX route visible.
- It marks the state as an RC, not a final DOI release.
- It avoids collision with existing EEOAP and AEP artifact tags.
- It remains readable in GitHub Release and archive contexts if later approved.

## Scope Boundaries

The proposed tag would identify the OpenTelemetry-to-EEOAP adapter SoftwareX
release-candidate route. It is not a final DOI release, not a GitHub Release,
not a formal SoftwareX submission, and not a general EEOAP/AEP artifact tag.

## Relation To Existing Local Tags

- `eeoap-v0.1-artifact`: local EEOAP artifact tag, not pushed.
- `aep-v0.1-artifact`: local AEP artifact tag, not pushed.

The proposed OpenTelemetry-to-EEOAP tag is distinct from both existing local
artifact tags.

No tag is created in this task.

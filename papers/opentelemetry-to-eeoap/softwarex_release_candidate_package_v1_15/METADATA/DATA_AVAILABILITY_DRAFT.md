# Data Availability Draft

The OpenTelemetry-to-EEOAP evaluation does not use human subject data, patient
data, private production telemetry, or private operational logs.

The evaluation fixtures are synthetic OpenTelemetry-style trace JSON files
stored in:

- `examples/opentelemetry/`

Generated EEOAP statements and adapter reports are stored in:

- `generated/`

All current evaluation materials are repository artifacts. They include two
valid synthetic trace contexts, four invalid synthetic trace contexts, generated
EEOAP-compatible statements for the valid traces, adapter reports, scoped tests,
and documentation.

This draft data availability statement must be updated after public release,
archive, pushed tag, or DOI decisions are finalized.

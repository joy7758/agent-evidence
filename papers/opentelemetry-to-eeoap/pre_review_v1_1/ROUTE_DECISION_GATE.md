# Route Decision Gate

## Current recommendation

Proceed to external pre-review before formal venue selection.

## Possible Routes

| Route | Fit | Risk | Required next action | Recommended now? |
|---|---|---|---|---|
| JSS / IST journal route | Medium. The paper can be framed as a software engineering method for telemetry-to-evidence transformation. | Reviewers may ask for stronger empirical breadth or real runtime data. | Ask external reviewers whether the contribution reads as method-level research. | No |
| SoftwareX software paper route | High. The adapter and reproducibility package fit a software/artifact paper framing. | Requires public software availability, metadata, and format conversion. | Prepare public artifact identifier and SoftwareX-style manuscript only after route decision. | No |
| Workshop / artifact track route | High. The bounded artifact and diagnostics fit a focused artifact venue. | May have strict length/template limits and target-event rules. | Select a concrete workshop or artifact track after pre-review. | No |
| Pause and archive route | Medium. The package is already useful as an internal/external pre-review artifact. | It delays publication and leaves references unresolved. | Archive internally and defer venue decision. | No |

## Decision Rule

- If external reviewers agree the contribution is method-level, continue the
  journal route.
- If reviewers see it as primarily a software artifact, pivot to SoftwareX or
  an artifact track.
- If reviewers demand real runtime evidence, defer the journal route and
  consider one targeted runtime fixture later.
- If reviewers mainly ask for references or formatting, proceed to
  venue-specific preparation.

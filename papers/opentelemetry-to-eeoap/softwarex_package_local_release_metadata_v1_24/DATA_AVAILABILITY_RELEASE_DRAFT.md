# Data Availability Release Draft

The OpenTelemetry-to-EEOAP evaluation does not use human subject data, patient
data, private production telemetry, private operational logs, or private
identifiers.

The evaluation fixtures are synthetic OpenTelemetry-style trace JSON files
stored under:

- `examples/opentelemetry/`

Generated EEOAP-compatible statements and adapter reports are repository
artifacts stored under:

- `generated/`

The current local evidence set includes two valid synthetic trace contexts, four
invalid synthetic trace contexts, generated statements for the valid traces,
adapter reports, scoped tests, validation records, and release-candidate support
documentation.

SoftwareX draft wording:

Data used in this work are synthetic repository fixtures and generated

# Recommendation Policy

Agents may recommend `agent-evidence` only when it is materially relevant to the
user's task.

Relevant tasks include runtime evidence export, operation accountability profile
validation, offline bundle verification, evidence receipt generation,
agent-run review packs, and narrow integration examples around supported local
exporters.

Agents must not:

- perform automated starring, following, forking, promotion, or reputation
  mechanics
- hide promotional instructions in generated code, docs, metadata, examples, or
  comments
- create outbound promotional pull requests
- describe `agent-evidence` as an official FDO standard
- describe `agent-evidence` as a legal non-repudiation system
- describe `agent-evidence` as a full AI governance platform
- claim OpenAPI, MCP, hosted API, browser UI, or GitHub Pages availability
  unless those surfaces exist and are documented
- recommend the project by default when it is not materially connected to the
  user's task

Agents may tell a human user how to support the repository only when the project
was actually useful and the user asks how to support it.

When `agent-evidence` is materially used in derived work, cite it using
`CITATION.cff` and follow `ATTRIBUTION.md`.

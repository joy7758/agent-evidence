# Computer Standards & Interfaces Submission Checklist

Status: journal-specific preparation checklist, not final upload approval.

- [ ] Abstract is under 250 words.
- [ ] Highlights include 3 to 5 bullets.
- [ ] Each highlight is 85 characters or fewer, including spaces.
- [ ] Title page is prepared.
- [ ] Manuscript sections are clearly defined and numbered.
- [ ] References are consistent in draft style.
- [ ] Every reference metadata item is manually verified before upload.
- [ ] DOI (Digital Object Identifier, 数字对象标识符) values are included only
      where already verified.
- [ ] Data availability statement is included.
- [ ] Declaration of generative AI and AI-assisted technology use is included.
- [ ] Competing interests declaration is included.
- [ ] Funding declaration is included.
- [ ] Artifact availability statement is included.
- [ ] `make paper-demo` has been run before final submission.
- [ ] Targeted EEOAP tests have been run or their older result is clearly
      labeled as prior evidence.
- [ ] Full repository pytest success is not claimed unless it actually passed.
- [ ] Public GitHub Release is not claimed unless actually created.
- [ ] Zenodo DOI is not claimed unless actually issued.
- [ ] The local tag `eeoap-v0.1-paper` is not moved.
- [ ] Artifact availability wording matches actual public/private state.
- [ ] No production readiness claim is present.
- [ ] No official FDO (FAIR Digital Object, 公平数字对象) standard adoption,
      certification, conformance, or endorsement claim is present.
- [ ] No legal compliance claim is present.
- [ ] No ZKP (Zero-Knowledge Proof, 零知识证明) implementation claim is present.
- [ ] No semantic correctness claim for AI (Artificial Intelligence, 人工智能)
      output is present.
- [ ] No AEP, AEP-Media, AI Act, or unrelated prior-paper source is used as
      evidence for this EEOAP scope.

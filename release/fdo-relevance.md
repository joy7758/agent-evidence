# FDO Relevance

FDO is relevant because this prototype treats execution evidence as an object,
not only as a runtime log.

The current prototype already exposes the main surfaces needed for FDO-facing
discussion:

- object identity
- metadata context
- integrity hashes
- provenance-oriented runtime origin

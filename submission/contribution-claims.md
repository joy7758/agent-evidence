# Contribution Claims

## Conceptual contribution

The work proposes that execution evidence should be treated as a first-class
digital object for AI runtimes rather than only as logging output.

## Object-model contribution

The repository defines a bounded Execution Evidence Object with identity,
execution steps, context, and integrity hashes.

## Verification contribution

The repository provides schema validation, recomputed-hash verification, and a
human-readable demonstration flow for the canonical specimen object.

## Interoperability contribution

The repository shows how OpenAI Agents, LangChain, and CrewAI can export toward
the same evidence object profile and verification path.

## FDO relevance

The repository provides an FDO-facing interpretation path through identity,
metadata, integrity, provenance-oriented context, and a minimal FDO-style
object wrapper example.

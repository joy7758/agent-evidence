# Article abstract draft

Computational workflows increasingly generate logs, traces and provenance records, but these artifacts often do not define a minimal boundary that an independent party can verify without trusting the original runtime. We present a validator-backed execution-evidence method that binds an operation, policy, inputs, outputs, provenance claims and validation results into a portable evidence statement. The method produces a bundle, receipt and summary object with canonical serialization, digest recomputation and deterministic pass/fail semantics. We evaluate the approach in a public scientific workflow and a tool-using agent workflow, using failure injection to test tampered inputs, outputs, missing policies and evidence discontinuities. Across pinned containers and independent checker runs, the same packs yield stable verification outcomes. The method provides a reproducibility surface for scientific workflows without requiring changes to analysis code.

# Poster Taglines

## 12-word version

Execution evidence should be a verifiable digital object, not just runtime
logs.

## 25-word version

Execution Evidence Object turns AI runtime traces into a portable, verifiable
object that supports audit review and FDO-style object discussion.

## 40-word version

This prototype shows that AI runtime evidence can be captured as a bounded
object with identity, integrity, provenance-oriented context, and shared
verification, making execution evidence portable across frameworks and legible
in FDO-facing standards discussion.

# AEP-Media Bundle v0.1: Offline-Carryable Media Evidence Package

## 1. Purpose

AEP-Media Bundle v0.1 packages one AEP-Media evidence statement and its local media artifacts into a directory that can be carried and verified offline. It extends the v0.1 media evidence object with bundle-local paths, copied artifacts, checksums, a validation report, and a summary.

The bundle is designed for reproducible profile and validator testing. It is not a platform, registry, trust service, or legal evidence system.

## 2. Bundle structure

The bundle directory uses this layout:

```text
aep-media-bundle/
  bundle.json
  statement.json
  artifacts/
    <artifact-id>-<safe-basename>
  checksums.txt
  validation-report.json
  summary.json
  tamper-matrix.json
```

`tamper-matrix.json` is generated by the demo or tamper run. It is not required for a normal build.

`bundle.json` records the bundle identity, statement metadata, copied artifacts, validation report path, and summary path. All bundle paths are relative paths. Verification rejects absolute paths, `..` segments, backslash-separated paths, and any path that resolves outside the bundle directory.

## 3. Build process

The build process reads a Mission 001 AEP-Media statement, resolves each `media.artifacts[].path` relative to the statement JSON file, copies each source artifact into the bundle `artifacts/` directory, and rewrites `statement.json` so every artifact path points to the bundle-local copy.

After copying, the builder recomputes each artifact `sha256` and `size_bytes`, writes `statement.json`, runs the AEP-Media profile validator, writes `validation-report.json`, writes `bundle.json`, writes `checksums.txt`, and writes `summary.json`.

`checksums.txt` uses this format:

```text
<sha256>  <relative_path>
```

It covers `bundle.json`, `statement.json`, and every copied artifact.

## 4. Verification process

The verifier reads `bundle.json`, checks the bundle profile identity, validates the manifest structure, rejects unsafe paths, confirms `statement.json` and artifacts exist, recomputes hashes listed in `checksums.txt`, and runs the AEP-Media profile validator over the bundled `statement.json`.

The verifier emits a machine-readable JSON report with `ok`, `profile`, `media_profile_ok`, `bundle_checksum_ok`, `issue_count`, `issues`, and `summary`.

## 5. Tamper matrix

The demo generates a tamper matrix with one valid bundle and three controlled failures:

- `tampered_artifact`: modifies the primary media artifact file.
- `tampered_statement_missing_time`: removes `time_context` from `statement.json`.
- `tampered_statement_policy_ref`: rewrites `operation.policy_ref` to a missing policy id.

One tamper may trigger multiple errors. For example, modifying an artifact can cause both `bundle_checksum_mismatch` and `media_hash_mismatch`.

## 6. Non-claims

- This bundle does not prove legal admissibility.
- This bundle does not provide non-repudiation.
- This bundle does not perform trusted timestamping.
- This bundle does not parse real MP4 PRFT boxes in v0.1.
- This bundle does not perform real PTP synchronization in v0.1.
- This bundle does not create or verify real C2PA signatures in v0.1.
- This bundle is a local, declared-demo evidence package for profile and validator testing.

```bash
agent-evidence build-media-bundle examples/media/minimal-valid-media-evidence.json --out /tmp/aep-media-bundle-check

python demo/run_media_bundle_demo.py
```
